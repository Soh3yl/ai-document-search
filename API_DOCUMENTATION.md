# مستندات رابط برنامه‌نویسی (API Documentation)

سیستم جست‌وجوی هوشمند و پرسش‌وپاسخ، یک Endpoint (نقطه پایانی) یکپارچه برای دریافت سوالات و ارسال پاسخ‌ها در اختیار توسعه‌دهندگان (کلاینت‌ها) قرار می‌دهد.

---

## ۱. جست‌وجوی هوشمند اسناد (Semantic Search API)

این سرویس، پرسش کاربر را دریافت کرده، با استفاده از الگوریتم `TF-IDF` مرتبط‌ترین اسناد را می‌یابد و سپس به کمک `LangChain` (یا سیستم Fallback) یک پاسخ متنی تولید می‌کند. این سرویس مجهز به سیستم `Cache` برای پاسخ‌دهی سریع به سوالات تکراری است.

* **آدرس (URL):** `/api/search/`
* **متد (Method):** `POST`
* **نوع محتوا (Content-Type):** `application/json`

### بدنه درخواست (Request Body)
درخواست باید شامل یک آبجکت JSON با کلید `question` باشد:

```json
{
  "question": "جنگو چیست و چه مزایایی دارد؟"
}
```

### پاسخ موفق (Success Response - 200 OK)
در صورت موفقیت، سرور یک آبجکت JSON شامل پاسخ هوش مصنوعی، لیست اسناد مرتبط و وضعیت کش را برمی‌گرداند:

```json
{
  "generated_answer": "🤖 پاسخ هوش مصنوعی واقعی:\nجنگو یک فریم‌ورک قدرتمند پایتون است که...",
  "results": [
    {
      "id": 1,
      "title": "معرفی یک فریم‌ورک محبوب",
      "score": 0.85,
      "content_snippet": "این ابزار قدرتمند به برنامه‌نویسان کمک می‌کند تا..."
    }
  ],
  "cached": false
}
```

> **نکته:** فیلد `cached` نشان می‌دهد که آیا این پاسخ به صورت زنده پردازش شده (`false`) یا به دلیل تکراری بودن سوال، از حافظه موقت خوانده شده است (`true`).

### خطاهای احتمالی (Error Responses)

**۱. خطای 400 (Bad Request):** زمانی رخ می‌دهد که کلید `question` در بدنه درخواست ارسال نشده باشد.

```json
{
  "error": "لطفاً پرسش خود را در قالب {\"question\": \"...\"} ارسال کنید."
}
```

**۲. خطای 405 (Method Not Allowed):** زمانی رخ می‌دهد که کلاینت به جای `POST` از متدهای دیگر (مثل `GET`) استفاده کند.

```json
{
  "error": "فقط درخواست‌های POST مجاز هستند."
}
```

**۳. خطای 500 (Internal Server Error):** خطاهای پیش‌بینی‌نشده در سمت سرور یا دیتابیس.

```json
{
  "error": "متن خطای سرور..."
}
```
